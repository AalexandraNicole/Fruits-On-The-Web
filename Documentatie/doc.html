Product Scope

FrOW is an educational web game focused on fruits and vegetables, designed to promote healthy eating habits among primary school students. The application will feature interactive gameplay, educational content delivery, and user engagement features. Key features will include quiz-based challenges, informational resources, progress tracking, and user profiles.

The primary objective is to create an interactive platform that promotes learning about healthy eating habits among primary school students. By incorporating fruits and vegetables into gameplay and educational content, the application seeks to foster awareness and understanding of nutrition and dietary choices from an early age.

Overall Description

Product Perspective

FrOW is positioned as a standalone web application developed using HTML5, CSS, and JavaScript. It will integrate various components and interfaces to deliver an immersive educational experience. The application will be accessible via web browsers on desktop and mobile devices, providing a seamless user experience across platforms.

Product Functions

FrOW will offer a range of functions to engage users in interactive learning activities. Key functions include user authentication, gameplay features such as quizzes and challenges, educational content delivery through text, images, and videos, user engagement features such as leaderboards and progress tracking, and administrative functionalities for managing user accounts and content.

User Classes and Characteristics

FrOW will cater to primary school students, educators, and parents, each with distinct characteristics and requirements. Primary school students will engage with the application through gameplay and educational content, while educators will use the application to supplement classroom teaching with interactive resources. Parents will have access to monitoring and tracking features to support their children's learning progress.

Operating Environment

FrOW will operate within a web environment, compatible with modern web browsers such as:

    Communications Interfaces

    The educational web gaming platform requires several communication functions to operate effectively and securely:

    HTTP(S) Communication

    Protocol: HTTP/HTTPS
    Purpose: Facilitates communication between the user's web browser and the platform's web server.
    Standards: Utilizes HTTP/HTTPS protocols for data transfer between the client and server, ensuring compatibility with web browsers.
    Security: HTTPS is employed to encrypt data transmitted between the client and server, ensuring confidentiality and integrity of user data.
    Message Formatting: Data exchanged between client and server adheres to standard HTTP message formats, including headers and payloads.

    API Communication

    Protocol: HTTP/HTTPS
    Purpose: Enables communication between the platform and external APIs for data retrieval and updates.
    Standards: Utilizes GraphQL APIs over HTTP/HTTPS for interfacing with external services.
    Security: API requests are made over HTTPS to ensure data confidentiality and integrity during transmission.
    Message Formatting: API requests and responses conform to the specified API endpoint formats, typically JSON or XML payloads.

    Web Socket Communication

    Protocol: WebSocket
    Purpose: Supports real-time communication between the platform and users for interactive features such as chat or notifications.
    Standards: Utilizes the WebSocket protocol for bidirectional communication between the client and server.
    Security: WebSocket connections may be secured using SSL/TLS encryption to prevent unauthorized access or tampering of data.
    Message Formatting: WebSocket messages adhere to the WebSocket frame format, allowing for efficient and low-latency communication between client and server.

    Email Communication

    Protocol: SMTP
    Purpose: Facilitates communication with users via email for account verification, password resets, and notifications.
    Standards: Utilizes SMTP for sending emails.
    Security: SMTP communication may be secured using SSL/TLS encryption to protect sensitive email content during transmission.
    Message Formatting: Email messages conform to MIME standards, allowing for the inclusion of text, HTML, and attachments as needed.

    RSS Feed Communication

    Protocol: HTTP/HTTPS
    Purpose: Provides access to the leaderboard of top players as an RSS feed.
    Standards: Utilizes HTTP/HTTPS to fetch the RSS feed data from the server.
    Security: RSS feed URLs may be accessed over HTTPS to ensure data integrity and confidentiality.
    Message Formatting: RSS feed entries adhere to the RSS specification, containing metadata such as title, description, and link.
    These communication interfaces enable the educational web gaming platform to interact with users, external APIs, and services securely and efficiently, enhancing the overall user experience and functionality of the platform.
